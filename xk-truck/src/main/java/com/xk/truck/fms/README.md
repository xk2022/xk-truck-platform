# ✅ FMS 是什麼？

**FMS = Fleet Management System**
中文一般稱：**車隊管理系統**

---

# 🎯 為什麼我們在這裡要用 FMS？

因為你現在處理的範圍是：

| 要管理的對象      | 具體內容               |
| ----------- | ------------------ |
| 車輛（Vehicle） | 車牌、車種、狀態（可用/維修/忙碌） |
| 司機（Driver）  | 名稱、駕照類型、在線狀態       |
| 車輛與司機關係     | 指派 / 解除 / 當前正在由誰駕駛 |
| 車隊運作        | 後續會擴充派工、維修記錄、里程、油耗 |

這些全部統稱為：

> **車隊管理（Fleet Management）**

因此模組命名為 **`xk-truck.fms`** 是精準且國際通用的做法。

---

# 🧱 在你的系統中，FMS 扮演的位置

```
UPMS  → 管帳號/權限
ORDER → 管訂單的生命週期
FMS   → 管人（司機）+ 車（車輛）+ 車隊狀況
DISPATCH（下一步）→ 把訂單指派給車/司機
```

一旦 **FMS** 做好，你才能開始做 **派工（Dispatch）**。

---

# 🌍 FMS 是業界標準用語

物流業 / 貨運業 / 集運業 / 通運業 / 卡車運輸產業 **都用這個字**。

例子：

| 國際系統         | 中文說法          |
| ------------ | ------------- |
| Trimble FMS  | 車隊管理平台        |
| Samsara FMS  | 車隊稽核 / 行車監控系統 |
| Geotab Fleet | 車隊 IoT 運輸管理   |

所以你用這個縮寫是 **專業且通用的**。

---

# 📌 簡單一句話定義（給客戶也能聽得懂）

> **FMS 就是讓公司知道「哪些車、哪些司機、現在在哪、可不可以派」的系統。**
